## Daily Workflow
1. Pull the latest changes from main.
2. Create a feature branch.
3. Complete your assigned task.
4. Commit frequently using meaningful commit messages.
5. Push your branch.
6. Open a Pull Request.
7. Wait for review before merging.
## Commit Convention
- Added business understanding notebook
- Cleaned dataset and handled missing values
- Implemented descriptive statistics
- Created sales dashboard visualizations
- Integrated Milestone 1 deliverables
## Repository Rules
- Never push directly to `main`.
- Always create a feature branch.
- Submit changes through a Pull Request.
- Write descriptive commit messages.
- Keep notebooks runnable from top to bottom.
- Store large datasets outside the repository when practical.
- Update documentation whenever project structure changes.
